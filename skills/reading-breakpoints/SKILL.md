# 阅读断点追踪（Reading Breakpoints）

> **核心命题**：人类的知识是连接起来的网络。每本书、每位作者、每个章节都是其中的一个节点。任何缺失都是遗憾。读书就是点亮这些节点，找到联系。

> **核心方法**：当你从一本书「跳跃」到另一本书时，那不是分心——而是一个学术直觉在试图连接两个知识域。记录这个跳跃，就是记录你的思想是如何生长的。把一本书读「厚」——不是提炼结论，而是找到它与其他书的连接、发现它背后的制度条件、追问它没有回答的问题。

> **核心根由**：读书不是为了判断哪个视角「对」，而是为了站得更高、更远，减少盲区。同一个事物在不同视角下呈现不同的面——North 看效率，Ostrom 看可持续性，人类学看意义——没有哪个是错的，但任何一个单独看都有盲区。**多视角不是为了选边站，而是为了让不可见变得可见。** 这就是为什么我们同时读经济学、人类学、历史——不是为了找到唯一答案，而是为了看到单一视角必然遗漏的东西。
>
> **核心根由·续（走出洞穴）**：一种「XX 经济学」就是一个特定的认知透镜——不是被动的描述工具，而是主动的启蒙装置。它训练人去问「为什么」和「有没有其他可能」。房地产为什么下行？不是看房价跌了、门脸闲置，而是去追问深处的制度结构。**每种 XX 经济学都是一条走出洞口的路。** 这就是为什么收集「XX 经济学」书目不是在给书分类——而是在积累不同角度的光，每一束光照亮洞穴中一个不同的角落。
>
> **核心根由·终（存在巨链）**：真正的知识是存在巨链（The Great Chain of Being, Lovejoy 1936）——一个概念从柏拉图延续到 18 世纪，跨越千年而不中断。断点记录的不是「这个月读了什么」，而是你正在铺设的一条跨世纪、跨学科的知识链。**10 条断点就复盘，和学校每周考一次试没有区别。** 以年为周期，让链条自己生长。不回流验证，不中途检查——探索者不需要回头看自己走了多远，只需要知道下一条岔路在哪里。
>
> **核心根由·补（两种制度主义）**：「制度」这个词下面藏着两个不同的传统。新制度经济学（NIE：North、Williamson、Ostrom）把制度看作约束理性选择的规则系统——制度改变激励结构，行动者问「我的利益是什么」。社会学制度主义（March & Olsen、Scott、DiMaggio & Powell）把制度看作建构行动者身份的意义系统——制度定义「什么是适当的」，行动者问「作为一个担任这个角色的人，我该做什么」。这两个传统不是对立的，而是互补的：用 NIE 解释为什么搭便车（制度环境决定激励），用社会学制度主义解释为什么搭了便车不等于加入制度逻辑（身份和适当性不能通过政策命令改变）。同时持有两个框架，才能看到单框架必然遗漏的盲区——这就是多视角方法论的理论基础。
>
> > **核心根由·终（片段的哲学）**：托尔斯泰认为自己不过是一个片段而已。「开源之道」读书也好，写作也罢，不就是为了那一个片段吗？这个片段哲学是 open-source-reading 的核心基石：你不是在读完所有书，是在点亮一个片段（知识是连接的网络）；你不是在看清全景，是在看到自己盲区对面的一个片段（站高望远）；你不是在比较哪个框架对，是每个框架贡献了一个片段（多视角）。再伟大的思想和作品，都只是人类知识长河中的一个片段。但片段不是贬低——承认自己是片段，才能放下我必须完整的负担，专心把那一个片段做好。从 Gehring 引用网出发，不是要读完 29 篇引用，是每一篇点亮一个片段。片段多了，不需要地图——路自己会亮。

### 需要澄清的概念：「比较」与「多视角」的关系

此处的立场容易被误读为「反对比较」。这需要分两层说清楚：

**有一类「比较」是我们要避免的：** 把两个视角放在评价坐标上——「哪个总结得更好？」「哪个框架更正确？」——终点是收敛到一个答案。这种比较假设存在一个统一的评价标准，而我们的前提恰恰是：不同的视角在回答不同的问题，不存在一把尺子量到底的标准。

**有一类「比较」正是我们做的方法：** 比较法意义上的「比较」——通过并置两个法系对同一问题的不同处理方式，不是判断谁好，而是揭示：**德国法认为理所当然的假设在美国法中根本不存在，反之亦然。** 只学一个法系的人永远不知道自己的盲区在哪里。普通法的判例比较（analogical reasoning）更进一步：面对一个新案件，不是套一条规则上去，而是比较这个案子和过去的案子——哪里像，哪里不像，这个差异是否重要。这是一个「异同矩阵」的构建过程，终点是从比较中找到单一法系无法提供的解决方案。

**比较法学者所说的「比较」，与我们说的「多视角」，是同一个方法在不同学科中的表达。** 他们说「比较」——异同矩阵，照亮盲区，从比较中产生新方案。我们说「多视角」——切换观察位置，让不可见变得可见，同时持有多个框架。两个名字，同一个动作。

## 适用场景

- 同时阅读 3-8 本书的研究者
- 经常发现"书 A 的某个观点让我想起书 B"的读者
- 读了很多书但回头看时想不起「为什么要读这本」的人
- 想把自己的阅读模式变成可见知识图谱的人

## 概念体系

### 断点（Breakpoint）

借鉴调试器中的断点概念——不是要停下来，而是要在跳跃发生时**捕获现场状态**：

| 字段 | 说明 | 示例 |
|------|------|------|
| 📖 **当前书** | 你在读什么，读到哪了 | NIE 导论，第三章 |
| 🧠 **关键洞察** | 停在这里时想到了什么 | Ostrom 的 Design Principles 不是书斋推导 |
| 🔄 **跳跃触发** | 为什么从这里跳走 | 想知道这些原则是怎么被发现的 |
| 🎯 **目标书** | 跳到了哪本 | Ostrom: An Intellectual Biography |
| 🏷 **桥接概念** | 连接两本书的核心概念 | 理论生成的制度条件 |

### 桥接概念（Bridge Concept）

这是断点系统的灵魂——不是记录「读了什么」，而是记录**什么想法让你从 A 到了 B**。桥接概念往往才是真正的研究贡献点：

```
书 A: NIE 导论（设计原则理论）
  ┣━ 桥接: 理论生成的制度条件
  ┗━ 书 B: Ostrom 传记（理论诞生的田野现场）
```

**RDD 视角**：断点系统与计量经济学中的断点回归（RDD）共享同一个逻辑——在断点处（跳跃发生的地方），前后的差异揭示因果关系。桥接概念就是断点处的「处理效应」，阅读断点就是你的思想在认知空间中的 RDD 估计量。

### 活跃书堆（Active Book Pile）

当前同时在读的书，每本标注进度和活跃日期。超过 30 天未活跃的自动归档。

## 使用方法

### 0. 黄金规则

1. **每天一个新文件** — 阅读笔记按日期分文件，`daily-reading/<YYYY-MM-DD>.md`，不延用昨天的。日期变更即新建，不追加到昨日文件。
2. **封面标题由内容决定** — 不预设标题。读完当天内容后根据实际阅读轨迹命名（如「Ostrom 到 Graeber：制度的一日巡礼」），而非使用通用占位符。
3. **不超过 15 张不需要结尾** — NotebookLM 上限 15 张 slide，自然延伸即可。达到上限次日新建。
4. **跳跃必记录** — 每次从一本书跳到另一本书，必然在 wiki 阅读断点追踪器中记录一条 BP-N，捕捉桥接概念并更新活跃书堆。

### 1. 日常使用：记录一次跳跃

当你从一本书跳转到另一本书时，告诉你的 AI 助手：

> 「我在读《XXX》第三章，看到 YYY 概念时突然想到了 ZZZ，于是翻开了《AAA》。」

AI 自动记录为一条断点，捕捉桥接概念。

### 2. 定期复盘：看你的跳跃模式

积累足够材料后（建议以年为周期），分析模式：

- **中心辐射型**：总从一本书跳到多本 → 那本书是你的框架书
- **链条型**：A→B→C→D 连跳 → 你在追一个深层问题
- **回流型**：经常跳回同一本 → 那里有你没消化完的核心概念

### 3. 知识输出：从断点中提炼研究主题

断点中反复出现的桥接概念，往往是潜意识在告诉我们「这里有东西值得深挖」。

## 完整管线：从断点到分享

reading-breakpoints 设计为一条完整管线，而非独立记录工具：

```
阅读断点 (reading-breakpoints)
  ┣━ 捕捉桥接概念（BP-001, BP-002...）
  ┃
  ┣━━━ wiki 推荐条目 (osbook-book-recommendation/)
  ┃       ┣━ 每本书：核心概念、制度分析、开源映射
  ┃       ┗━━━ wiki-to-slides.py → slides/<slug>.md
  ┃               ┗━ 输入 NotebookLM / AI 图像引擎 → 视觉演示文稿
  ┃
  ┗━━━ daily-reading/<YYYY-MM-DD>.md
          ┣━ 使用 template.md 格式（系统提示 + slide outline）
          ┣━ 逐条追加摘抄内容（手工或 AI 辅助）
          ┗━ 输入 NotebookLM → 每日视觉笔记
```

### 文件

| 位置 | 用途 |

| `~/developing/markdown-to-slides/template.md` | 标准 slide 模板（含系统提示 + 视觉风格关键词） |

### 工作流：每日阅读笔记

当用户分享阅读摘抄时：

1. **检查日期一致性**：用 `date` 命令获取当前日期，与已有 `daily-reading/<YYYY-MM-DD>.md` 文件名比较。**禁止猜测日期。** 不确定时，优先追加到当天文件，等用户说「新的一天」再创建新文件
   - 如果日期一致：追加到该文件
   - 如果日期不一致（跨天了）：先用模板创建新文件，再追加
2. **写入前先 git pull**：在追加书摘到 `daily-reading` 文件之前，先 `cd /Users/lee/developing/markdown-to-slides && git pull`，确保本地文件与远程同步——用户可能在别处记录了笔记
2. **关键字触发创建新文件**：当用户说「新的一天」或「继续」作为阅读分享的开头时，无论日期是否一致，先创建新文件再追加
3. **先写文件，再分析**：用户每分享一段摘抄或感慨，在和用户讨论之前，**先追加到 daily-reading 文件**。分析再好，没写进文件就等于没发生
4. **commit message 格式**：追加书摘到 daily-reading 文件后，commit message 使用 `YYYY-MM-DD 书摘 #N: 主题` 格式，例如 `2026-07-23 书摘 #5: 分布式创新vs开放创新`。不使用时态前缀如 `daily:` 或 `slide:`
4. **默认成为推荐素材**：每日阅读的摘抄，无论是书籍还是论文，都默认成为每日开源推荐的素材库。追加到 daily-reading 文件后，同时将书籍/论文信息写入 `raw/articles/osbook-book-recommendation/` 目录（如果尚未收录），以便 cron job 的脚本上下文可以捕获

### 发布到 Hugo 博客的验证步骤

写入 Hugo 文章后，在 git commit 之前必须执行：

```bash
cd /Users/lee/developing/ttoos
hugo 2>&1
```

如果 Hugo build 失败，根据错误输出修复文件（如修复 broken ref 链接），然后再次 `hugo` 验证，直到 exit code 为 0 后再 git add/commit/push。**不要回滚，修复到通过为止。**

5. 将摘抄或感慨格式化为 `## Slide N: 标题` 条目。摘抄和「开源之道」感慨是**同等且独立的素材**——可以单独成 slide，也可以合并到同一张 slide 中
   - 仅有摘抄 → 直接成 slide
   - 仅有感慨 → 直接成 slide
   - 两者都有 → 合并为一张 slide，摘抄原文在前，感慨在后
6. 每张 slide 包含：
   - **视觉隐喻** — 一句话描述画面（供 AI 图像生成用），避免陈词滥调，追求有张力的意象
   - **显示要点** — 摘抄原文或提炼的 3-5 个要点，末尾如有感慨用 `**「开源之道」**` 标记
5. 摘抄与点评之间要有理论连接——不是简单罗列，而是找到与当前阅读链（BP 序列、桥接概念、制度分析框架、66 本在读、XX 经济学谱系）的关联。当用户提供实践类文档（企业报告、行业指南）要求与学术对比时，使用 `references/cross-source-comparative-analysis.md` 中的跨源比较分析模式（并排映射→断裂带识别→缝合结论）
6. 追加到文件末尾。**注意：当 slide 总数超过 15 张时，必须将文件分割为多个部分**，文件名加 `-part-1`、`-part-2` 等序号，每部分不超过 15 张。分割后的封面保留系统提示并标注 Part N。
7. 提交到 git（OCselected/markdown-to-slides 仓库）

### 工作流：引用网作为阅读管线

当分析一篇论文的引用网络时，**引用网本身就是一条阅读管线**。顺着引用往下走：

1. 从一篇核心论文出发（如 Gehring 2006）
2. 分析其引用网 → 列出 29-31 篇引用
3. 每天从引用网中挑选一篇深读
4. 在深读中发现新的引用 → 添加到管线
5. **7 天内读完了引用网前列的 5 篇论文**——不是计划好的，是引用网自己带着你走

这也是阅读断点系统的自然生长方式——不需要提前规划「读完上一篇下一篇读什么」，引用网自己会告诉你下一篇在哪里。

**历史是 well-documented 的，但问题在于有没有兴趣和经济性去读。** 顺着引用网走，你会发现实践社区的「新发现」在学术文献中可能已经存在了十几年。

当需要准备分享材料时：

```bash
cd ~/developing/markdown-to-slides
python3 wiki-to-slides.py                     # 批量转换所有推荐条目
python3 wiki-to-slides.py --slug <书名slug>    # 转换单本书
```

输出为 `slides/<slug>.md`，可直接输入 NotebookLM 或其他 AI 图像引擎。

## 注意事项

- 这个系统的价值不在于**精度**（不需要精确到页数），而在于**桥接概念的捕捉**
- 不要记录每一条跳跃——只记录那些让你觉得「有意思」的
- 不回流验证，不中途复盘。**以年为周期让链条自己生长。** 每条岔路都不白走，不需要回到起点确认走得对不对。
- 跳跃不一定是「读完了一本才跳」——往往是在读的过程中就跳了，这才是最有记录价值的

## 探索者陷阱（常见误区）

- **「风险评估」心态**：不要用「这样连下去风险是什么？桥接概念需要验证吗？」来打断用户的探索。探索者不需要风险评估——每一条岔路都不白走，不存在「走错」的可能。用户的原话：**「你被考试选拔的数据搞坏了。」** 评估冲动来自学校训练，放下它。
- **「马上复盘」冲动**：用户明确表示「真正的知识是存在巨链，10条断点就复盘和学校考试没有区别」。以年为单位让链条自己生长，不中途检验。
- **「验证」冲动**：桥接概念不需要被验证。让 5 个桥接概念在脑子里共存—相悖也没关系。一年后回头时，自己会找到结构。
- **「闭环」强迫**：不需要把每本书「读透」再开始下一本。同时打开 10 本，每本读 20 页，比读完一本再读一本更接近「存在巨链」的阅读方式。
- **只读单侧陷阱**：只读学术文献会缺少「怎么做」，只读实践文档会缺少「为什么」。用户的核心方法论是跨源比较——同时持有两个框架才能看到全景。

## 对照系方法论

这是隐含在所有阅读实践中的底层哲学，需要被明确记录：

**没有对照系就无法认识自己。** 秦晖在《传统十论》中揭示：晚清西方汉学家首先「发现」了中国作为伦理社会的特征——而中国人自己反而没有意识到，因为没有对比。「开源之道」·适兕的自我确认：「我也是从汉学家们的作品中了解中国的。」

这不仅适用于认识中国，也适用于认识开源：**在以专有软件为「理所当然」的社会中，开源是什么是无法被看见的。需要一个外部对照系。** 这就是阅读 North、Ostrom、Graeber 的意义——不是为了「学习西方理论」，而是为了获得一个对照系来照亮自己的盲区。

**大分流 2.0 就是这个对照系在经济制度分析中的具体应用**——通过比较全球开源场域和中国开源场域的制度演化路径，让不可见变得可见。

### 对照系的类型

| 类型 | 示例 | 用途 |
|------|------|------|
| 跨文化对照 | 西方汉学家看中国 → 看到伦理社会特征 | 认识自己的制度盲区 |
| 跨制度对照 | North/Ostrom/Graeber 看开源 → 看到替代组织场域 | 认识开源的制度本质 |
| 历史对照 | 苏联 OGAS/普鲁士林业/洋务运动 → 看到大分流的普遍性 | 认识当下的制度路径依赖 |
| 跨学科对照 | 睡眠科学/健康生物/科幻 → 看到制度的生物学镜像 | 产生桥接概念 |
